# startup-techritual

Static catalogue site for **startups.techritual.com** — display all published independent app / SaaS / tool submissions from `gmail-auto-techritual` Firestore `article_submissions` collection.

- **Stack**: Astro 4 (static SSG) + Cloudflare Pages
- **Data**: build-time fetch from Firestore REST (公開 read，無 auth)
- **Refresh**: GitHub Actions cron 每週一 00:00 UTC + manual trigger
- **AdSense**: 繼承 techritual.com publisher account（auto-ads via `<head>` script）
- **Cost**: $0 (Cloudflare Pages 免費 unlimited bandwidth + GitHub free tier)

## 本地開發

```bash
npm install
npm run dev          # http://localhost:4321
npm run build        # 產出 dist/ static HTML
npm run preview      # 預覽 dist/
npm run snapshot     # 手動刷新 public/data/snapshot.json
```

## 部署機制

```
Author commit                       Cron weekly
       │                                  │
       ▼                                  ▼
 GitHub repo  ◄─── snapshot.json refresh (Actions)
       │
       ▼ webhook
 Cloudflare Pages
       │ npm run build (calls firestore.js → live fetch with snapshot fallback)
       ▼
 startups.techritual.com (CF edge CDN)
```

每次 `git push` → Cloudflare Pages 自動 build + deploy。每星期一 GitHub Actions cron 跑 `snapshot` script，update `public/data/snapshot.json` 後 commit + push，自動觸發 CF Pages re-deploy。

## 數據流

1. Astro build 時 `src/pages/index.astro` 呼叫 `fetchPublishedSubmissions()`
2. 嘗試 fetch Firestore REST（10 秒 timeout）
3. 成功：parse + filter（`status==='published'` + `wordpressPostUrl` 非空）+ 寫入 `public/data/snapshot.json` 做下次 fallback
4. 失敗：讀取 bundled snapshot.json
5. 渲染靜態 HTML（zero client-side data fetch）

## DNS + Cloudflare Pages 設定

依次執行：

1. **GitHub**: 推送呢個 repo 上 `Stoneip/startup-techritual`
2. **Cloudflare Pages**:
   - Dashboard → Workers & Pages → Create → Pages → Connect to Git
   - 揀 `Stoneip/startup-techritual` repo
   - Build command: `npm run build`
   - Build output: `dist`
   - Environment variables: 唔需要（Firestore public read）
   - Deploy
3. **Custom domain**: Pages project → Custom domains → Set up → `startups.techritual.com`
4. **DNS**: Cloudflare DNS dashboard for techritual.com:
   - 加 `CNAME startups → <project>.pages.dev`（Cloudflare Pages 會話畀你具體 hostname）
   - Proxy status: Proxied（橙色雲）
5. **SSL**: 自動，等 5-10 分鐘 Let's Encrypt provision
6. **Verify**: 開 `https://startups.techritual.com/`

## Manual snapshot refresh

如果想立即拎最新數據（唔等到星期一 cron）：

- GitHub Actions tab → "Refresh catalogue snapshot" workflow → "Run workflow"

或者本地：

```bash
npm run snapshot
git add public/data/snapshot.json
git commit -m "chore(snapshot): manual refresh"
git push
```
